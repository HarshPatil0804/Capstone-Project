[0:20 — WHAT IT IS]
AquaGuard is a multi-agent environmental health application built using the Google Agent Development Kit 2.0. It belongs to the "Agents for Good" track, helping citizens report tap water pollution and get immediate EPA-standard safety recommendations while automatically mapping public safety issues.

[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the workflow architecture. Every report begins at the Security Checkpoint, where inputs are sanitized for privacy and protected from adversarial prompt injections. From there, clean reports move to our first specialized LLM agent, the Report Analyzer. The Report Analyzer identifies the specific contaminant and queries an MCP server to retrieve official EPA water safety standards. If the Analyzer determines the threat is high severity, the workflow pauses at the Human Approval node, prompting an administrator to manually approve volunteer mobilization and hotspot mapping. Once approved, or if the severity is low, the Action Planner LLM agent takes over to log the contamination to the hotspot map and suggest step-by-step mitigation plans for the neighborhood. Finally, the terminal node displays the complete safety plan and logging confirmation.

[3:00 — SECURITY & MCP]
Under the hood, AquaGuard leverages a local FastMCP server which connects the agents to a file-based database for logging hotspots and managing volunteers. Additionally, the Security Checkpoint runs full PII scrubbing to redact emails and phone numbers, and prints detailed JSON audit trails of all system decisions.
